## 設計脈絡(v1 → v4 演化)

本工作坊設計經過多輪迭代:

- **v1**:5 個提案並列(A 對照組 / B 逆向 / C 四劇場 / D 慢讀 / E Meta-Loop)
- **v2**:user 點出真正的對立不是「壓縮 vs 還原」而是 **「在原稿上做筆記 vs 在新頁面開摘要」**(不同物種)
- **v3**:user 三個答案後,B 退出(那是建造者聽眾),A/C/D 都還在
- **v4**:user 拒絕「選 1 或合併」框架,要求**把 A/C/D/E 圈出的有效片段組裝成一個 coherent 設計**,並補入三道哲學提問作為敘事骨

## 三道哲學提問(現場與招生敘事貫穿)

整場 2 小時不是流程教學,是**三道學習哲學問題的展開**,
「在原稿上做筆記 vs 在新頁面開摘要」是回答這三道問題的中間工具。

### Q1 · 我們什麼時候會成為學生?

> 我們因為**想要獲得知識、獲得能力、獲得技術,並且存放在自己的本人身上**,
> 打算用這樣的結果去面對與應對未來生活的某一種互動的反應準備。

**目的**:讓觀眾意識到,daily 使用 AI 並不等於正在成為學生。把大腦外
包給 AI 的時候,你只是 AI 的下游介面,而不是有意識地建立學生身份的人。

### Q2 · 怎麼才能把「獲得」給落實?

> 「獲得」在掌握程度有什麼樣的分類?是否需要將所有的學習都昇華成為反射?
> 當學習的結果試圖要逐漸逼近內化、用起來覺得信手捻來的時候,你該做的
> 練習與複習的方式與工具是什麼?

**目的**:讓觀眾從「我是學生」(身份)推進到「**獲得的層級**」(知道
→ 會用 → 信手捻來 → 反射),並質問現在的 AI 工作流是否真的能幫你走完
這條路徑。

### Q3 · 時間有限的情況下,AI 該怎麼賦能而不是替代學生?

> 在透過『語言模型』這樣新的工具中,我們該用來賦能、協作與加速在學習
> 這件事情的內化 roadmap 上,**是否有一種全新的體驗能夠來試試看?**

**目的**:用 Meta-Loop 預告(今天會錄影、變成下週寄回的好學生筆記)
實際示範「全新的體驗」就是觀眾今天剛剛經歷的。

## 觀眾與痛點

- **跨部門、跨產業、跨公司**,**「懶惰是不論部門,不論公司,不論行業的」**
- **Daily AI users**(每天用 ChatGPT/Claude/Gemini/NotebookLM)
- **已經把大腦外包**:把 Karpathy LLM Wiki raw text 塞進 Obsidian
  做三維知識串接 — 這不是他在讀,是 AI 替他記住
- **失去「自己讀」的肌肉**

> **不是工具建造者**(內訓人員 / 學習長 / 教育總監)— 那是另一場
> 工作坊的設計方向(v2-v3 的 B 提案,本次定案不採用)。

## 核心觀念樞紐:摘要 vs 筆記 不同物種

| | 摘要(NBLM / Gemini / ChatGPT TL;DR) | 筆記(傳統人類 / 好學生筆記) |
|---|---|---|
| 位置 | **新開一頁** | **在原稿上** |
| 形式 | 條列、節錄、目錄式 | 重點劃線、邊欄註記、重複強調、立場註寫 |
| 結構 | 像「目錄」 | 像「閱讀軌跡」 |
| 關係於原稿 | 替代 | 增厚 |
| 學習者讀什麼 | 讀那份新摘要(原稿被擱置) | 讀原稿(摘要等於沒做) |
| 對 Q2「獲得」的貢獻 | 知道(最淺) | 從知道一路通到反射 |

**好學生筆記 = 把 AI 帶到「在原稿上做筆記」的位置**,不是「另開一頁做摘要」的位置。

- Step 1-2(SRT → cleaned.md):**還原原稿**,讓筆記有「原稿」可以做
- Step 3 知識補充:**在原稿位置加旁註**,不是另開一頁
- Step 4 立場注入:**在原稿位置加視角**,不是另開一頁

這對應到工作坊的論點:**這不是 NBLM 失敗了,是 NBLM 做的事根本不是「筆記」**。

## 兩小時節目單

| 時段 | 主題 | 內容 | 時長 |
|---:|---|---|---:|
| 0:00 - 0:10 | **開場 + Q1 質問 + Meta-Loop 揭示** | 揭示「今天會被錄影 + 一週後寄你一份好學生筆記」。直球質問 **Q1:我們什麼時候才會成為學生?** 留 90 秒沉默讓觀眾自己想 | 10 min |
| 0:10 - 0:25 | **物種 reframe(對照震撼)** | 「你最近一份『AI 幫你做的筆記』長什麼樣?是另開一頁的目錄,還是在原稿上的標記?」並列投影:**共同錄音**(講師預備好的中文演講)的 NBLM 摘要版 vs 好學生筆記版,讓觀眾在 60 秒內**自己看出**這是不同物種 | 15 min |
| 0:25 - 0:55 | **慢讀體驗:在原稿上做筆記是什麼感覺** | 全體用**印刷紙本**讀同一段好學生筆記(5-8 千字、講師預備的素材),在邊欄畫線、寫旁註、寫立場 — **這是 Q1 的身份證明**(只有學生才會做這件事) | 30 min |
| 0:55 - 1:10 | **Q2 對話:獲得的層級與內化** | 觀眾分享紙本邊欄的兩三個註記。**Q2 引出**:剛剛你做的事,對你「獲得」的程度是哪一級?知道?會用?信手捻來?反射?— 講師簡介內化 roadmap(知道 → 應用 → 反射) | 15 min |
| 1:10 - 1:30 | **動手 1:碎形切片 + Step 1-2** | 「**要在原稿上做筆記,前提是原稿不能破碎**」 — 觀眾用自帶錄音跑 Step 1-2(轉錄 + cleaned.md),體驗 95-105% 字數帶的紀律 | 20 min |
| 1:30 - 1:55 | **動手 2:在原稿上加旁註 + 加橋** | 「**在原文難處旁邊加旁註,不是另寫一份簡單版**」(知識降維 Step 3) + 「**在原文陌生詞彙處,加上你自己領域的橋**」(認知橋接 Step 4) — 觀眾在剛剛的 cleaned.md 上實作 Step 3 旁註 + Step 4 立場;親身體驗 AI 留在原稿上的力量 | 25 min |
| 1:55 - 2:00 | **Q3 引線 + 結語 + Meta-Loop 預告** | **Q3:時間有限的時候,AI 該怎麼賦能而不是替代學生?**「你今天做的,就是答案的一部分。一週後,你會收到今天這場的好學生筆記 — **這份筆記本身就是這個答案的證明**。」 | 5 min |
| **合計** | | | **120 min** |

### 明確拒絕的元素(已剔除)

- B 提案的「拆解一份好學生筆記的縫合痕跡」 — user 原話「這塊不是不重要,是已經被做好了,所以該講的是原理不需要拆解」 → 改為用慢讀讓觀眾自己讀出原理,不當場 reverse-engineer
- A 原本的「你還記得 YouTube 摘要嗎?」這種 memory-test 開場(對 daily AI users 不可靠)
- 松尾研日文當主對比素材(觀眾沒共同經驗就 OK 不了)
- 自帶錄音當主對比素材(對比階段用共同錄音,自帶留到 hands-on)

## 必備資源清單

### 講師端前置(招生前 → 上課前 2 週)

1. **共同錄音預備** — 一段 30-60 min 的中文演講(自錄 / 公開演講皆可),預跑兩個版本:
   - NBLM 摘要版(直接上傳到 NBLM,匯出摘要)→ 存到 `materials/common-recording/<event>/notebooklm-summary.md`
   - 好學生筆記版(跑完 Step 1-4 流程)→ 存到 `materials/common-recording/<event>/cleaned.md`
   - 原音檔 → `materials/common-recording/<event>/audio.mp3`(gitignored)
2. **紙本印刷母版** — 從上述好學生筆記版選 5-8 千字最有展示力的一段,排版成「邊欄留白給觀眾寫字」的紙本 → `materials/paper-handout/<event>/source.md` + `layout.pdf`
3. **招生海報文案** — 用本檔的招生宣言 + 三道提問做主視覺 → `materials/recruitment/poster-source.md`
4. **報名表內容** — 標明「請帶自己的錄音(15-30 min 演講 / 訪談 / 會議)」+ 提供範本下載 → `materials/recruitment/registration-form.md`

### 現場端

- 投影設備(對照投影 + Meta-Loop 揭示)
- 觀眾筆電(或共用一台跑 hands-on)
- 麥克風 + 錄影機(Meta-Loop 必需)
- 紙本印刷 10-15 份(每人一份 + 備用)
- 馬克筆 / 螢光筆(讓觀眾在紙本上做筆記)
- 計時器(每段時間管理)
- Wi-Fi + Groq + Gemini / Claude / NBLM 帳號(API key 預存好)

### 課後端(Meta-Loop 履行,SLA 見 W5)

1. **48 小時內**:錄影檔轉 mp3 → 跑 Step 1-2 → 講師親自做 Step 3-4
2. **一週內**:把 cleaned.md(或 enhanced.md / notes_<identity>.md)**直接寄給觀眾**
   — email body 貼純 markdown 或附 .md 檔案,**不做網頁**
3. **二次推播**:這場的 cleaned.md 截圖 + 觀眾在紙本上的旁註照片做成社群素材
